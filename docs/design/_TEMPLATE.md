## Guidelines for Design Docs

**✅ DO Include:**

- User-facing behavior and flows
- Problem statement and user benefits
- High-level feature overview
- Visual/UX design notes
- Implementation status
- Future enhancement ideas

**❌ DON'T Include:**

- Specific code implementations
- Detailed API contracts
- TypeScript interfaces
- Component file structures
- Database schemas
- Testing code examples
- Step-by-step technical instructions

**Remember:** Specs track the technical HOW. Design docs explain the user-focused WHAT and WHY.
