# mobile-050: Video Analysis Calibration Tests

**Status:** In Progress
**Depends on:** mobile-046 (video form analysis), mobile-049 (additional video metrics)

## What This Covers

End-to-end calibration tests that run the video analysis pipeline against real stock footage and validate outputs against a manifest of expected results. The existing unit tests (92 Vitest) use synthetic sine-wave landmark data. These tests close the gap: they verify the full pipeline produces sensible metrics on actual gym video, and they create regression anchors so future algorithm changes don't silently break.

Six royalty-free Pexels videos are already downloaded in `test-videos/`. A `manifest.json` describes each video's metadata and expected outputs. The tests run the analysis pipeline (frames → landmarks → metrics), compare results to the manifest, and fail on regressions once calibrated.

### Manifest Structure (`test-videos/manifest.json`)

Each video entry has:
- `expected` — predicted camera angle, rep count range, which faults/metrics to check
- `actual` — `null` until populated by a calibration run
- `calibrated` — `false` until a human verifies `actual` values visually

### Pipeline Under Test

```
video file → extractFramesFromVideo(uri, duration, 4fps)
           → { frames: PoseFrame[], fps }
           → analyzeVideoFrames(frames, fps, lift)
             → interpolateEmptyFrames()
             → assembleAnalysis()
               → bar path extraction (wrist landmarks)
               → rep detection (Y-periodicity peak finding)
               → per-rep: angles, depth, drift, velocity, faults, grading
           → VideoAnalysisResult { reps[], fps, cameraAngle, analysisVersion }
```

Key entry points in `modules/video-analysis/`:
- `application/analyze-video.ts` — `extractFramesFromVideo()`, `analyzeVideoFrames()`
- `lib/metrics-assembler.ts` — `assembleAnalysis()` (analysisVersion 3)
- `lib/fault-detector.ts` — thresholds and fault detection
- `lib/bar-path.ts`, `lib/angle-calculator.ts`, `lib/rep-detector.ts`, `lib/depth-detector.ts`, `lib/bar-velocity.ts`, `lib/rep-tempo.ts`
- `lib/fatigue-signatures.ts` — cross-rep fatigue analysis
- `lib/butt-wink-detector.ts`, `lib/stance-width.ts`, `lib/hip-shift.ts` — squat-specific
- `lib/elbow-flare.ts`, `lib/pause-quality.ts` — bench-specific
- `lib/hip-hinge-timing.ts`, `lib/bar-shin-distance.ts` — deadlift-specific
- `lib/lockout-stability.ts` — all lifts

## Key Thresholds (from `fault-detector.ts` + `metrics-assembler.ts`)

These are the thresholds the tests should validate against:

| Fault | Lift | Threshold | Severity |
|---|---|---|---|
| `above_parallel` | squat | hip Y ≤ knee Y at bottom | critical |
| `excessive_lean` | squat | forward lean > 55° | warning |
| `bar_drift` | all | drift > 0.03 normalized (~7.3cm) | warning |
| `back_rounding` | deadlift | hip angle deviation > 15° mid-pull | warning |
| `incomplete_lockout` | deadlift | hip angle < 170° at top | info |
| `butt_wink` | squat | hip angle drop > 10° in < 200ms at bottom | warning |
| `elbow_flare` | bench | flare > 80° or < 30° | warning |
| `early_hip_shoot` | deadlift | hip velocity crossover < 30% of pull | warning |
| `bar_away_from_shins` | deadlift | wrist-knee distance > 5cm | warning |
| `unstable_lockout` | all | hip angle CV > 5% in last 10% of rep | warning |
