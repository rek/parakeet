# AI Coaching

LLM integration strategy, constraints, and prompt architecture.

---

## Models

| Purpose              | Model         | Timeout | Fallback       |
|----------------------|---------------|---------|----------------|
| JIT session          | gpt-4o-mini   | 5000 ms | Formula engine |
| Motivational message | gpt-4o-mini   | 5000 ms | Silent skip    |
| Cycle review         | gpt-5         | async   | None           |
| Challenge judge      | gpt-5         | 8000 ms | Silent skip    |
| Decision replay      | gpt-5         | async   | None           |

SDK: `@ai-sdk/openai` (Vercel AI SDK). Note: engine uses v3.x (`LanguageModelV2`), app uses v1.x (`LanguageModelV1`). Import `getJITModel`/`getCycleReviewModel` from `@parakeet/training-engine`. In production, calls are routed through a Supabase Edge Function proxy configured via `configureAIProxy()` at bootstrap.

**Source:** `packages/training-engine/src/ai/models.ts`

## JIT Strategy

Three strategies available:

| Strategy | Behavior |
|----------|----------|
| Formula  | Pure engine calculation (default). No LLM call. |
| LLM      | Full session generated by LLM with structured output. |
| Hybrid   | Formula + LLM in parallel. Compare results. |

### Hybrid Comparison

| Condition | Action |
|-----------|--------|
| Weight within 10% AND same set count | Use LLM result |
| Diverge > 15% | Surface both to user with formula override option |
| LLM timeout/error | Fall back to formula silently |

**Source:** `packages/training-engine/src/generator/hybrid-jit-generator.ts`

## LLM Constraints (JIT)

Hard limits enforced post-LLM response:

| Parameter          | Range         |
|--------------------|---------------|
| intensityModifier  | 0.40 - 1.20   |
| setModifier        | -3 to +2      |
| restAdjustments    | -60s to +60s   |
| rationale strings  | Max 5, 200 chars each |

If the LLM returns values outside these bounds, they are clamped.

**Source:** `packages/training-engine/src/ai/prompts.ts`

## Challenge Mode

Post-hoc evaluation of JIT prescriptions.

### Judge Review (fire-and-forget after JIT)

| Field       | Type    | Description                    |
|-------------|---------|--------------------------------|
| score       | 0-100   | Prescription quality rating    |
| concerns    | string[]| Issues identified              |
| strengths   | string[]| Good decisions noted           |

### Decision Replay (fire-and-forget after session completion)

Retrospective scoring: given the actual RPE/volume outcomes, how accurate was the prescription?

| Field          | Type    | Description                      |
|----------------|---------|----------------------------------|
| accuracy_score | 0-100   | How well prescription matched outcome |
| insights       | string[]| What the system could learn      |

**Source:** `packages/training-engine/src/ai/prompts.ts`, `challenge_reviews` + `decision_replay_logs` tables

## Motivational Messages

LLM-generated post-workout message on the completion screen.

### Context Provided

Session RPE, PRs achieved, current streak, biological sex, cycle phase, completion %.

### Rules

- Multiple sessions per day consolidated into a single message
- Timeout: 5000 ms, silent skip on failure
- Max length: not explicitly constrained (UI truncates)

**Source:** `apps/parakeet/src/modules/session/application/motivational-message.service.ts`

## Cycle Review

End-of-program AI analysis.

### Output Structure

| Field                     | Max Length | Content                              |
|---------------------------|-----------|--------------------------------------|
| overallAssessment         | 500 chars | Summary of cycle performance         |
| progressByLift            | per lift  | Rating + narrative                   |
| auxiliaryInsights         | --        | Most/least correlated exercises      |
| volumeInsights            | --        | Under/over-trained muscles           |
| formulaSuggestions        | --        | Priority + overrides                 |
| structuralSuggestions     | --        | For developer review                 |
| nextCycleRecommendations  | 2000 chars| Forward-looking guidance             |
| menstrualInsights         | 1000 chars| Optional, cycle-phase-specific       |

### Multi-Cycle Context

Up to 3 previous cycle summaries injected into the prompt to detect trends across cycles.

**Source:** `packages/training-engine/src/ai/prompts.ts`, `engine-012`
